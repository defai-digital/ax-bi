# Code of Conduct

AX BI is developed and maintained by DEFAI Private Limited. This code of
conduct applies to public project spaces managed for AX BI, including GitHub
issues, discussions, pull requests when invited, documentation comments, release
threads, and any DEFAI-managed community channel for the project.

AX BI is not accepting unsolicited public code contributions or pull requests.
The standards below still apply to bug reports, feature requests, product
feedback, reproducible issue reports, and any agreed contribution path.

## Expected Behavior

Participants are expected to:

- Be respectful and professional.
- Assume good intent where reasonable, while still taking responsibility for the
  impact of their words and actions.
- Keep discussions focused on the project, the issue being reported, or the
  product feedback being provided.
- Provide clear, reproducible technical details when reporting bugs.
- Respect maintainers' decisions about scope, roadmap, prioritization, and
  contribution policy.
- Avoid sharing private, confidential, proprietary, or credential-bearing
  information in public channels.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment, intimidation, personal attacks, or sustained disruption.
- Discriminatory language or behavior related to identity, background, ability,
  religion, nationality, gender identity, sexual orientation, age, or similar
  personal characteristics.
- Sexualized language, unwanted sexual attention, or explicit material in
  project spaces.
- Threats of violence or encouragement of harm.
- Publishing or threatening to publish private information without permission.
- Posting secrets, credentials, customer data, private logs, or confidential
  business information.
- Repeatedly reopening closed topics, bypassing moderation, or pressuring
  maintainers to accept unsolicited work.
- Misrepresenting AX BI, DEFAI Private Limited, Apache Superset, or the Apache
  Software Foundation.

## Reporting Issues

If you see behavior that violates this code of conduct, report it through the
project's available private contact path when one is listed by DEFAI Private
Limited. Do not include sensitive personal information, credentials, customer
data, or confidential logs in a public issue.

For security vulnerabilities, follow the repository's security policy instead
of opening a public issue.

## Enforcement

Maintainers may take actions they consider appropriate for the project space,
including:

- Asking a participant to edit or remove content.
- Closing, locking, hiding, or deleting comments, issues, discussions, or pull
  requests.
- Limiting further participation in project spaces.
- Reporting platform abuse through the hosting provider.

Enforcement decisions are based on the behavior, its impact, and the risk to
the project and its participants.
